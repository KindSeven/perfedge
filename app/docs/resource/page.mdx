# 静态资源优化

> 网页的静态资源往往是最能影响 FCP 指标的。比如打开某个音视频网站，当前页面中有很多较大的图片，受限于网络速度，图片加载会出现白屏状态，这无论对于用户体验还是性能指标都是不友好的行为。因此对静态资源进行优化，可以显著提升网页的性能。

---

## 目录

- **Web 图片：**

  - 使用合适的格式：根据需要选择合适的图片格式，例如 JPEG、PNG、WebP 或 SVG，以提高图片质量和压缩率。
  - 图片懒加载：通过懒加载技术，只有在用户滚动到相应位置时才加载图片，减少初始加载时间。
  - 图像压缩：使用在线工具或库（如 ImageOptim、TinyPNG）来压缩图片，减小文件大小。

- **Web 字体：**

  - 字体子集：只加载页面中使用到的字符，避免加载整个字体文件。
  - 使用现代字体格式：如 WOFF2，提供更好的压缩和加载性能。
  - 延迟加载字体：使用 `font-display: swap` 属性来避免字体加载时的空白。

- **代码压缩：**

  - 压缩 HTML、CSS 和 JavaScript：使用工具（如 Terser、CSSNano）来删除多余的空白、注释和未使用的代码。
  - 使用 Tree Shaking：在构建过程中移除未使用的模块，以减小代码体积。

- **分包策略：**

  - 代码分割：使用动态导入和懒加载技术，将代码分成更小的块，按需加载。
  - 第三方库分离：将第三方库单独打包，并利用浏览器缓存，避免每次加载都下载相同的库。

- **缓存策略：**
  - 设置合理的缓存头：利用 HTTP 头部的缓存控制策略，设置 `Cache-Control` 和 `Expires` 头，提升资源的缓存效率。
  - 使用 Service Workers：通过 Service Workers 实现更灵活的缓存策略，支持离线访问和资源更新。
  - 定期更新缓存：根据版本号或哈希值更新资源，确保用户始终获得最新版本的文件。

通过这些优化措施，可以显著提升网页的加载速度和用户体验，从而提升网站在搜索引擎中的排名。
